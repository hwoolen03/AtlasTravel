The HTML code includes a header, a button to open the login form, and a hidden login form. The form includes text inputs for the username and password, two buttons to submit and cancel the form, and two buttons to sign in with Google and Facebook. The JavaScript code handles the visibility and submission of the login form, and the sign-in functionality with Google and Facebook. Step 4: Create the Server-Side Code Now that we have the client-side code ready, letâ€™s create the server-side code to handle the login requests. We will use Node.js and Express to create the server. First, create a new directory for the project and navigate to it: mkdir social-login cd social-login Next, create a new Node.js project and install the required dependencies: npm init -y npm install express body-parser cors Create a new file named server.js and add the following code: // server.js const express = require('express'); const bodyParser = require('body-parser'); const cors = require('cors'); const app = express(); app.use(cors()); app.use(bodyParser.json()); app.post('/save-login', (req, res) => { const { username, password } = req.body; console.log(`Username: ${username}, Password: ${password}`); res.send('Login successful'); }); app.post('/google-login', (req, res) => { const { token } = req.body; console.log(`Google Token: ${token}`); res.send('Google login successful'); }); app.post('/facebook-login', (req, res) => { const { token } = req.body; console.log(`Facebook Token: ${token}`); res.send('Facebook login successful'); }); app.listen(3000, () => { console.log('Server running on http://localhost:3000'); }); This code sets up an Express server with three POST routes to handle the login requests: /save-login - Handles the form submission with the username and password. /google-login - Handles the Google sign-in request. /facebook-login - Handles the Facebook sign-in request. The server logs the received data and sends a success message back to the client.
